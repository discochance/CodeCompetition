CONVAY'S GAME OF LIFE

Benutzung

Öffnen der Simulation

Um die Simulation zu öffnen, müssen Sie einen Browser installiert haben, z.B. Firefox oder Chrome.
Aus Leistungsgründen empfehle ich Firefox 55+, aber jeder andere Browser mir Javascript unterstützung sollte ebenfalls funktionieren.
Öffnen Sie die im Quellverzeichnis, welches Sie durch entpacken der zip-Datei erhalten, erhaltene index.html Datei.
Ihr PC sollte dabei automatisch einen Browser verwenden, falls nicht, wählen Sie selbst einen.

Erstellung eines Spielfeldes

Vor dem Starten des Algorithmus, muss ein Spielfeld festgelegt werden. Mit den Eingabefeldern für die Anzahl der Zeilen und Spalten in dem Header
lässt sich die größe des Spielfeldes regeln, wobei änderungen direkt übernommen werden.

Das Eingabefeld für die Zellenbreite legt fest, wie groß die Zellen dargestellt werden.

Vorbereitungen für den Start

Bevor der Algorithmus gestartet werden kann, muss ein Anfangsbestand gegeben sein, da ein komplett tote Welt nicht wiederbelebt wird.
Dafür stehen Ihnen 2 Möglichkeiten zur Verfügung:

  1. Die Schaltfläche zum zufälligen wählen des Status jeder Zelle, wobei die Chancen 50/50 stehen, dass die Zelle lebt
  2. Das manuelle Wählen von lebenden Zellen. Hierbei tippen Sie auf die Zelle die ihren Status ändern soll
Lebende Zellen werden grün dargestellt.

Start des Algorithmus

Nachdem lebendige Zellen existieren, kann der Algorithmus gestartet werden. Dazu müssen Sie nur auf die "Start" Schaltfläche klicken.
Sobald Sie den Algorithmus starten, wird die erste Generation berechnet und dargestellt. Der Generationen-Counter im rechten oberen Eck zeigt dabei an,
welche Generation gerade angezeigt wird.
Über die "Nächste Generation"-Schaltfläche können Sie wieder die nächste Generation berechnen.

Falls Sie die Generationen automatisiert berechnen lassen wollen, klicken Sie einfach auf "Automatisch". Die Website berechnet dann automatisiert die nächsten Generationen
wobei das Textfeld mit der Beschriftung "Gen. pro Sek." angibt, wie viele Generationen pro Sekunde berechnet werden sollen.

ACHTUNG! Diese Angabe ist nur der vom Script vorgesehene Wert! Durch unterschiedliche Leistungsstärken von PC's kann es durchaus vorkommen,
dass nicht mehr als 2 bis 3 Generationen pro Sekunde berechnet werden können!

Alle Änderungen der Geschwindigkeit werden sofort umgesetzt.
Zum stoppen der automatisierten Berechnung wieder einfach die Schaltfläche anklicken.

Soll der Aktuelle Bestand verändert werden, können Sie einfach auf "neu beginnen" klicken

Zusatzfunktionen

Antiwelt erstellen

Durch einen klick auf die Schaltfläche "Antiwelt erstellen" sterben alle lebenden Zellen und alle toten Zellen werden wiederbelebt.
Es wird also genau die Gegenwelt erschaffen.

Welten importieren/exportieren

Durch einen rechtsklick irgendwo auf der Seite öffnet sich das Kontextmenü. In diesem können Sie über die Schaltfläche "Welt exportieren"
Die gesamte Welt exportieren. Dafür öffnet sich ein neuer Tab dessen Textinhalt Sie kopieren und speichern müssen, z.B. unter dem Namen "Meine_Welt.wrld", wobei die Dateiendung irrelevant ist,
der Einfachheit halber kann man die Datei auch als Textdatei speichern.

Über die Schaltfläche "Welt importieren" können Sie eine Beispiel-Welt oder eine vorher gespeicherte Welt importieren. Dazu kopieren Sie einfach den Dateiinhalt der gespeicherten Datei
in das angezeigte Textfeld und klicken auf "importieren". Dabei wird Ihre aktuelle Welt überschrieben!
Einige Beispielwelten befinden sich im Quellordner, darunter

  • die Gosper Glider Gun, eine der ersten entdeckten "Guns"
  • die Gosper Glider Gun, in extra großer Welt, damit Sie sich nicht so schnell selbst zerstört
  • eine große leere Welt, perfekt zum beobachten komplexer, zufälliger Welten
  • zwei verschiedene Schwimmer, das sind Konstrukte die Sich in unendlich eine Richtung bewegen können
  • ein lightweight spaceship, ein spezieller Schwimmer
  • sowie ein Pulsar der periode 3, ein Konstrukt welches immer wieder die selben Zustände durchspielt
Alle Beispielwelten sind als ".wrld" Datei abgespeichert, die Sie allerdings mit jedem Texteditor öffnen können. Sollte sich Ihr Texteditor weigern,
können Sie die dateien auch einfach in ".txt" Dateien umbenennen.

Zukunftsvorhersage

Wenn Sie die geneauen Vorgänge und Hintergründe von verschiedenen Konstrukten nachvollziehen möchten, ist das Vorhersage-Tool sehr nützlich.
Sobald der Algorithmus das erste mal gestartet wurde, löst das klicken auf eine Zelle nicht mehr das umschalten zwischen lebendig und tot aus,
sondern gibt eine Meldung aus, welchen Status die Zelle im nächsten Zyklus haben wird, und wieso.

Eine Beispielmeldung kann dann so aussehen: "Zelle bleibt leben, weil sie 2 oder 3 lebende Nachbarn(2) hat."

Dabei gibt die Zahl in Klammern den tatsächlich vorgefundenen Wert an lebenden Nachbarn an.

Verwerfen toter Welten

Erkennt der Algorithmus, dass keine lebende Zelle mehr existiert, dann wechselt er automatisch in den Start Modus, sodass der Spieler einen neue Startbestand wählen kann.

Regeln

Bei der Vorliegenden Version des Conways Game of Life handelt es sich um eine Version mit dem Regelsatz 23/3.
Dabei geben die Zahlen vor dem "/" an, wieviele lebende Nachbarn eine Zelle haben muss um am leben zu bleiben, und die Zahlen nach dem "/" wie viele lebende Nachbarn eine Zelle haben muss, um wiederbelebt zu werden.

Es gilt also:

  • Eine Zelle mit weniger als 2 lebenden Nachbarn stirbt, sozusagen aus Einsamkeit
  • Eine Zelle mit mehr als 3 lebenden Nachbarn stirbt, sozusagen aufgrund Überbevölkerung
  • Eine Zelle, die tot ist, mit genau 3 lebenden Nachbarn wird wiederbelebt
  • Eine Zelle mit 2 oder 3 lebenden Nachbarn, die selbst auch lebt, bleibt am Leben
Dies ist einer der üblichsten Regelsätze von Conways Game of Life, es sind aber auch viele andere Variationen bekannt.
Für den Rand des Spielfeldes gilt in diesem Fall, dass einfach auf der gegenüberliegenden Seite weitergemacht wird, wie als wäre das Gitter auf einer Kugel.
Andere Simulationen arbeiten mit einem unendlich großen Raum, der in jede Richtung unendlich weitergeht.

Zur Simulation

Die Simulation wurde von mir, Sebastian Bach, im Rahmen der CodeCompetition von it-talents.de im September 2017 entworfen und entwickelt.
Die Simulation funktioniert in purem HTML, CSS, Javascript und JQuery. Zur Erstellung dieser Webapp habe ich ca 10 Arbeitsstunden benötigt, weitere 5 zur Kommentierung und etliche weitere beim beobachten vieler faszinierender Welten.
